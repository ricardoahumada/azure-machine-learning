Exportar y desplegar un modelo Scikit-learn

1. Formatos para Exportar Modelos Entrenados de Scikit-learn

Una vez que hemos entrenado un modelo de machine learning utilizando Scikit-learn, es fundamental exportarlo en un formato adecuado para su uso posterior.

  • La exportacin permite guardar el modelo entrenado en un archivo, lo que facilita su reutilizacin, despliegue en produccin o comparticin con otros equipos.
  • Scikit-learn ofrece varias opciones para exportar modelos, cada una con sus propias caractersticas y casos de uso.

Formatos Comunes para Exportar Modelos

a. Pickle (.pkl)

pickle es una biblioteca estndar de Python que permite serializar objetos, incluidos modelos de Scikit-learn, en un archivo binario.

  • Ventajas:
    • Fcil de usar y compatible con cualquier objeto de Python.
    • Ideal para proyectos internos o prototipos rpidos.
  • Desventajas:
    • No es seguro para cargar modelos desde fuentes no confiables, ya que puede ejecutar cdigo malicioso durante la deserializacin.
    • Puede ser ineficiente para grandes modelos debido a su tamao.
  • Ejemplo de uso:
    import pickle
with open("model.pkl", "wb") as f:
    pickle.dump(model, f)

b. Joblib (.joblib)

joblib es una biblioteca optimizada para objetos grandes que contienen datos numricos, como los modelos de Scikit-learn. Es especialmente til para modelos que incluyen matrices NumPy.

  • Ventajas:
    • Ms eficiente que pickle para grandes modelos.
    • Soporta compresin, lo que reduce el tamao del archivo.
  • Desventajas:
    • Limitado a sistemas basados en Python.
  • Ejemplo de uso:
    import joblib
joblib.dump(model, "model.joblib")

c. ONNX (Open Neural Network Exchange)

ONNX es un formato abierto diseado para interoperabilidad entre diferentes frameworks de machine learning. Permite exportar modelos de Scikit-learn a un formato estndar que puede ser utilizado por otras herramientas o plataformas.

  • Ventajas:
    • Compatible con mltiples frameworks (TensorFlow, PyTorch, etc.).
    • Ideal para integrar modelos en pipelines multi-framework.
  • Desventajas:
    • Requiere conversin explcita usando herramientas como skl2onnx.
    • Algunas funcionalidades avanzadas de Scikit-learn pueden no ser compatibles.
  • Ejemplo de uso:
    from skl2onnx import convert_sklearn
from skl2onnx.common.data_types import FloatTensorType

initial_type = [('float_input', FloatTensorType([None, 4]))]
onnx_model = convert_sklearn(model, initial_types=initial_type)

with open("model.onnx", "wb") as f:
    f.write(onnx_model.SerializeToString())

d. PMML (Predictive Model Markup Language)

PMML es un estndar XML para representar modelos de machine learning. Es ampliamente utilizado en entornos empresariales para la interoperabilidad entre sistemas.

  • Ventajas:
    • Compatible con herramientas de terceros y sistemas empresariales.
    • Facilita la interpretacin y validacin del modelo.
  • Desventajas:
    • Complejo de implementar manualmente.
    • Requiere bibliotecas adicionales como sklearn2pmml.
  • Ejemplo de uso:
    from sklearn2pmml import sklearn2pmml
from sklearn2pmml.pipeline import PMMLPipeline

pipeline = PMMLPipeline([("classifier", model)])
sklearn2pmml(pipeline, "model.pmml", with_repr=True)

Consideraciones al Elegir un Formato

Al seleccionar un formato para exportar tu modelo, ten en cuenta los siguientes factores:

  • Interoperabilidad: El modelo ser utilizado en un entorno especfico o necesita ser compatible con mltiples frameworks?
  • Seguridad: El modelo ser cargado desde fuentes externas o no confiables?
  • Rendimiento: El modelo es grande y requiere optimizacin en trminos de tamao y velocidad de carga?
  • Entorno de Despliegue: El modelo ser utilizado en un sistema basado en Python o en una plataforma diferente?

2. Proceso de Despliegue de una App Flask que Usa un Modelo de Scikit-learn

El despliegue de una aplicacin Flask que utiliza un modelo de Scikit-learn implica varios pasos clave, desde el entrenamiento del modelo hasta su integracin en un entorno de produccin.

  1. Entrenamiento del Modelo: Se entrena un modelo de Scikit-learn utilizando datos histricos.
  2. Exportacin del Modelo: El modelo entrenado se guarda en un archivo (por ejemplo, .pkl o .joblib) para su uso posterior.
  3. Creacin de la API Flask: Se desarrolla una API Flask que carga el modelo y proporciona endpoints para realizar predicciones.
  4. Despliegue en Produccin: La API Flask se despliega en un entorno de produccin utilizando herramientas como gunicorn, Docker o servicios en la nube (Azure, AWS, etc.).

Entrenamiento del Modelo

En esta etapa, se entrena un modelo de Scikit-learn utilizando un conjunto de datos. El modelo puede ser cualquier algoritmo compatible con Scikit-learn, como regresin logstica, rboles de decisin, etc.

  1. Preparar los datos de entrenamiento.
  2. Dividir los datos en conjuntos de entrenamiento y prueba.
  3. Entrenar el modelo utilizando los datos de entrenamiento.
  4. Evaluar el modelo en los datos de prueba.

Exportacin del Modelo

Una vez entrenado, el modelo debe guardarse en un archivo para poder ser utilizado en la API Flask.

  1. Usar joblib o pickle para guardar el modelo en un archivo binario.
    import joblib
joblib.dump(model, "model.pkl")
  2. Verificar que el archivo exportado contiene el modelo entrenado.

Creacin de la API Flask

La API Flask es responsable de cargar el modelo exportado y proporcionar endpoints para realizar predicciones.

  1. Crear un archivo app.py con la lgica de la API Flask.
  2. Cargar el modelo exportado en la API.
    import joblib
model = joblib.load("model.pkl")
  3. Definir un endpoint /predict que reciba datos de entrada y devuelva predicciones.
    @app.route("/predict", methods=["POST"])
def predict():
    data = request.json
    features = np.array(data["features"]).reshape(1, -1)
    prediction = model.predict(features)
    return jsonify({"prediction": int(prediction[0])})

Despliegue en Produccin

Finalmente, la API Flask se despliega en un entorno de produccin para que pueda ser utilizada por aplicaciones o usuarios finales.

  1. Contenerizacin (Opcional): Crear un contenedor Docker para la API Flask.
    • Crear un archivo Dockerfile.
    • Construir la imagen Docker:
      docker build -t flask-api .
    • Ejecutar el contenedor:
      docker run -p 5000:5000 flask-api
  2. Despliegue en un Servidor:
    • Usar gunicorn para servir la API:
      gunicorn -w 4 app:app
    • O desplegar en plataformas en la nube como Azure, AWS o Heroku.
  3. Pruebas: Verificar que la API funciona correctamente enviando solicitudes HTTP.

3. Ejemplo

Instalacin de Dependencias

Antes de comenzar, asegrate de instalar las bibliotecas necesarias:

pip install scikit-learn flask joblib

3.1. Entrenamiento y Exportacin del Modelo

Entrenar un Modelo de Scikit-learn

Vamos a entrenar un modelo simple de clasificacin utilizando el conjunto de datos Iris.

from sklearn.datasets import load_iris
from sklearn.linear_model import LogisticRegression
import joblib

# Cargar datos
X, y = load_iris(return_X_y=True)

# Entrenar modelo
model = LogisticRegression(max_iter=200)
model.fit(X, y)

# Guardar el modelo en un archivo usando joblib
joblib.dump(model, "model.joblib")
print("Modelo guardado como 'model.joblib'")

Nota: Tambin puedes usar pickle para guardar el modelo:

import pickle
with open("model.pkl", "wb") as f:
    pickle.dump(model, f)

3.2. Crear una API Flask

Estructura del Proyecto

Crea una estructura de proyecto:

proyecto/

 model.pkl          # Modelo exportado
 app.py             # Cdigo de la API Flask
 requirements.txt   # Dependencias

Cdigo de la API Flask (app.py)

El siguiente cdigo implementa una API Flask que carga el modelo y realiza predicciones.

from flask import Flask, request, jsonify
import joblib
import numpy as np

# Inicializar Flask
app = Flask(__name__)

# Cargar el modelo
model = joblib.load("model.pkl")

# Definir la ruta para realizar predicciones
@app.route("/predict", methods=["POST"])
def predict():
    try:
        # Obtener datos JSON de la solicitud
        data = request.json
        features = np.array(data["features"]).reshape(1, -1)  # Convertir a formato adecuado
        
        # Realizar la prediccin
        prediction = model.predict(features)
        
        # Devolver la prediccin como respuesta JSON
        return jsonify({"prediction": int(prediction[0])})
    except Exception as e:
        return jsonify({"error": str(e)}), 400

# Ruta de prueba
@app.route("/", methods=["GET"])
def home():
    return "API Flask funcionando correctamente"

if __name__ == "__main__":
    app.run(debug=True)

Archivo requirements.txt

Asegrate de incluir las dependencias necesarias en un archivo requirements.txt:

Flask==2.3.2
scikit-learn==1.3.0
joblib==1.3.2
numpy==1.23.5

3.3. Ejecutar la API Flask

Instalar Dependencias

Desde la terminal, instala las dependencias:

pip install -r requirements.txt

Ejecutar la API

Ejecuta la API Flask:

python app.py

La API estar disponible en http://127.0.0.1:5000.

3.4. Probar la API

Usar curl para Probar la API

Puedes probar la API usando curl:

curl -X POST http://127.0.0.1:5000/predict \
-H "Content-Type: application/json" \
-d '{"features": [5.1, 3.5, 1.4, 0.2]}'

Respuesta Esperada

La API devolver una respuesta JSON con la prediccin:

{
  "prediction": 0
}

Usar Python para Probar la API

Tambin puedes probar la API desde Python:

import requests

# URL de la API
url = "http://127.0.0.1:5000/predict"

# Datos de entrada
data = {"features": [5.1, 3.5, 1.4, 0.2]}

# Enviar solicitud POST
response = requests.post(url, json=data)

# Mostrar la respuesta
print(response.json())

3.5. Despliegue en Produccin

Usar un servidor para Servir la API

Para desplegar la API en produccin, usar gunicorn:

pip install gunicorn
gunicorn -w 4 app:app

Contenerizar la API con Docker

Crea un archivo Dockerfile para contenerizar la API:

# Base image
FROM python:3.9-slim

# Set working directory
WORKDIR /app

# Copy files
COPY . .

# Install dependencies
RUN pip install --no-cache-dir -r requirements.txt

# Expose port
EXPOSE 5000

# Run the app
CMD ["gunicorn", "-w", "4", "app:app"]

Construye y ejecuta el contenedor:

docker build -t flask-api .
docker run -p 5000:5000 flask-api
